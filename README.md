# 🐧 KodeKloud Linux Administration Labs Portfolio

Chào mừng bạn đến với kho lưu trữ thực hành Quản trị Hệ thống Linux của mình! Đây là nơi mình ghi lại toàn bộ quá trình tự học, ghi chú câu lệnh cốt lõi và nhật ký xử lý sự cố (Troubleshooting) từ chuỗi **20 bài Lab Thực Hành chuyên sâu** trên nền tảng **KodeKloud Studio**.

Dự án này được xây dựng nhằm mục đích "vật chất hóa" các kinh nghiệm thực hành CLI thực tế, minh chứng cho năng lực quản trị, cấu hình mạng và bảo mật hệ thống Linux độc lập của bản thân.

## 🗂️ Sơ Đồ Phân Loại Lab & Kỹ Năng Hệ Thống

Toàn bộ 20 bài lab được mình phân loại khoa học thành 5 nhóm chuyên môn chính của một System Engineer/DevOps. Bạn có thể click vào từng thư mục dưới đây để xem chi tiết các câu lệnh và kịch bản giải quyết bài toán:

### 📂 [01. Commands & Core Concepts](./01-commands-and-concepts/)
*Tập trung vào tư duy hệ thống, thao tác tệp tin nâng cao và thành thạo môi trường dòng lệnh (CLI).*
- [x] **Linux Basic Commands 1** - Điều hướng, quản lý file và thư mục cơ bản.
- [x] **Linux Basic Commands 2** - Tập lệnh tìm kiếm, lọc dữ liệu nâng cao (`grep`, `find`, `wc`).
- [x] **Linux Core Concepts 1** - Kiến thức nền tảng về Kernel, Shell và Không gian người dùng (User space).
- [x] **Linux Core Concepts 2** - Cơ chế quản lý tài nguyên và kiến trúc hệ thống Linux.
- [x] **Linux Bash Prompt** - Tùy biến và tối ưu hóa môi trường làm việc của Terminal Shell.
- [x] **Linux vi Editor** - Chỉnh sửa file cấu hình hệ thống nhanh chóng qua CLI bằng Vi/Vim.

### 📂 [02. Package & Service Management](./02-package-and-services/)
*Quản lý vòng đời phần mềm, cài đặt dịch vụ và tự động hóa các tiến trình chạy ngầm.*
- [x] **Linux Package Management** - Cơ chế quản lý gói phần mềm trên các bản phân phối (APT/YUM).
- [x] **Linux RPM & YUM** - Cài đặt, cập nhật phần mềm thủ công và cấu hình Repo trên hệ điều hành họ RedHat.
- [ ] **Linux Cron Jobs** - Lập lịch tự động hóa các tác vụ định kỳ trong hệ thống.
- [ ] **Linux systemd Services** - Khởi tạo, quản lý và giám sát trạng thái các dịch vụ hệ thống ngầm.

### 📂 [03. Permissions & Security](./03-permissions-and-security/)
*Kiểm soát truy cập, bảo mật định danh người dùng và thiết lập tường lửa bảo vệ máy chủ.*
- [x] **Linux IPTables** - Thiết lập quy tắc tường lửa (Firewall rules) chặn/mở traffic mạng bảo vệ hệ thống.
- [ ] **Linux Account Management** - Quản trị tài khoản người dùng, nhóm (User/Group) và cấp quyền sudoers.
- [ ] **Linux Permissions & Ownership** - Quản lý phân quyền tệp tin và quyền sở hữu (`chmod`, `chown`, ACL).
- [ ] **Linux SSH & SCP** - Cấu hình truy cập máy chủ từ xa an toàn và truyền tải file bảo mật qua mạng.

### 📂 [04. Networking & Storage](./04-networking-and-storage/)
*Cấu hình các phân vùng lưu trữ vật lý/logic và thiết lập kết nối mạng diện rộng.*
- [ ] **Linux Networking** - Cấu hình IP, định tuyến, gateway và kiểm tra kết nối mạng.
- [ ] **Linux DNS** - Cơ chế phân giải tên miền và cấu hình phân giải trên Linux Host.
- [ ] **Linux Storage 1** & **Linux Storage 2** - Quản lý phân vùng đĩa cứng và lưu trữ nâng cao.
- [ ] **Linux Storage Filesystems** - Định dạng hệ thống tệp tin (Ext4, XFS) và cơ chế Mount/Umount dữ liệu.

### 📂 [05. Troubleshooting](./05-troubleshooting/)
*Kỹ năng tối quan trọng của SysAdmin: Đọc log hệ thống, khoanh vùng sự cố và khôi phục dịch vụ.*
- [ ] **Linux troubleshooting** - Phân tích nhật ký lỗi và khắc phục các sự cố thực tế của hệ điều hành.

## 🎯 Giá trị cốt lõi đút kết được từ Repo này
* **Kỹ năng thực hành 100% qua CLI:** Không phụ thuộc vào giao diện đồ họa (GUI), làm chủ hoàn toàn terminal.
* **Tư duy Troubleshooting:** Mỗi bài lab là một bài toán thực tế, giúp rèn luyện khả năng đọc hiểu thông báo lỗi, tra cứu log file và đưa ra phương án xử lý tối ưu.
* **Tài liệu hóa công việc:** Thói quen viết tài liệu (Documentation) rõ ràng, giúp đồng đội và hệ thống dễ dàng quản lý thông tin chuyển giao.
